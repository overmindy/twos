# Twos 2.0 数据库索引优化建议

为了保障在大规模用户下的查询效率，建议在微信云开发控制台为以下集合建立索引：

## 1. users 集合
*   `_openid`: 唯一索引 (默认已建立)
*   `isSearching`: 单字段索引 (用于匹配加速)

## 2. relationships 集合
*   `userA`: 单字段索引
*   `userB`: 单字段索引
*   `status`: 单字段索引
*   `[userA, status]`: 复合索引 (加速 getActiveRelationship)
*   `[userB, status]`: 复合索引 (加速 getActiveRelationship)

## 3. answers 集合
*   `relationshipId`: 单字段索引
*   `questionId`: 单字段索引
*   `privacy`: 单字段索引
*   `createTime`: 单字段索引 (倒序)
*   `[relationshipId, createTime]`: 复合索引 (加速时光长河分页)
*   `[privacy, createTime]`: 复合索引 (加速广场信息流)

## 4. messages 集合 (止戈室)
*   `relationshipId`: 单字段索引
*   `createTime`: 单字段索引 (正序，用于 Watch 同步)

## 5. album 集合
*   `relationshipId`: 单字段索引
*   `createTime`: 单字段索引 (倒序)

## 6. capsules 集合
*   `relationshipId`: 单字段索引
*   `openTime`: 单字段索引 (用于检查开启状态)
