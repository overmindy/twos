# 《两只(TWOS)》- 养成互动式情侣手札小程序 设计规格说明书

## 1. 项目愿景 (Overview)
《两只(TWOS)》是一款专为情侣或亲密关系设计的互动小程序。它不仅是一个交流工具，更是一本“由两个人共同书写的私人手札”。产品以“纸质书面”的美学风格为基调，通过“双盲解锁”、“养成植物”和“实时共振”等机制，将琐碎的日常互动转化为具有仪式感的情感资产。

## 2. 视觉系统 (Visual System) - "私人手札" (The Private Journal)
*   **材质层 (Texture)**: 背景采用带有细微纤维纹理的暖米色道林纸 (#FDFCF8)。
*   **元素层 (Elements)**:
    *   **线条**: 模拟钢笔手画感 (Rough Edges)，带有不规则的轻微抖动。
    *   **配色**: 墨黑 (#2F2F2F) 为主调，朱砂红 (#B22222) 用于印章/点缀，茶青 (#8A9A5B) 用于成长反馈。
*   **排版 (Typography)**: 全文强制使用衬线体 (如：思源宋体)。核心标题和部分正文支持竖排版 (Vertical Layout)。

## 3. 核心功能 (Core Features)

### 3.1 绑定机制：微信直连 (WeChat Seamless Linking)
*   **流程**: 一方发起邀请 -> 生成带参数的微信分享卡片 -> 另一方点击即自动确认绑定 -> 建立云数据库关联记录。
*   **反馈**: 绑定成功后，双方手机触发一次具有质感的“重-轻”震动。

### 3.2 每日一问：折叠便笺 (The Unfolding Memo)
*   **机制**: 双盲解锁 (Double-blind Reveal)。每天推送一个深度话题（如：未来、情感、假设）。
*   **交互**: 
    *   **未答题**: 首页展示一个折叠三次的纸片，封面盖有红色日期印章。
    *   **输入**: 点击纸片展开，呈现方格信纸。
    *   **解锁**: 双方均回答后，纸张缓慢摊平 (Unfold Animation)，文字像墨水渗入纸张般显现。
*   **呈现**: 左右双栏排版，中间虚线连接。

### 3.3 羁绊之树：博物志插画 (Botanical Illustration)
*   **视觉**: 首页背景为一副钢笔白描风格的植物插图。
*   **演变**: 互动（答题/共振）产出“墨水养分”。
    *   **成长**: 线条逐渐加密，出现淡彩水彩晕染。
    *   **枯萎**: 3天不互动，纸张颜色泛黄，线条变浅，出现水渍纹理。

### 3.4 心跳共振：墨印扩散 (Ink Resonance)
*   **交互**: 首页底部圆形印章按钮。
*   **反馈**: 长按时，指尖扩散出层层墨水晕染效果 (Canvas 实现)。
*   **震动**: 对方手机实时触发模拟心跳节奏的线性震动。

### 3.5 冲突缓冲：语气实验室 (The Ink Eraser)
*   **功能**: 利用 AI (大模型) 将带有负面情绪或攻击性的文字，润色为婉转、书面化的非暴力沟通表达。

## 4. 架构与路由 (Architecture & Routing)
*   **首页 (Home)**: 核心展示区（羁绊之树 + 每日一问卡片 + 指印按钮）。
*   **时光轴 (Archive)**: 像翻阅书籍一样查看往年的对话记录，支持日期索引。
*   **气象站 (Resonance)**: 显示物理距离、对方当地天气、对方当前状态。
*   **我的 (Settings)**: 用户设置、解除绑定、导出记录。

## 5. 技术方案 (Technical Architecture)
*   **前端**: 微信小程序原生框架 (WXML/WXSS/JS)。
*   **动画**: CSS3 Transition + Canvas (墨水晕染/纸张物理模拟)。
*   **后端**: 微信云开发 (Cloud Development)。
    *   **云数据库**: 实时同步 (Watch) 双方状态。
    *   **云函数**: 定时任务推送话题、AI 润色接口集成。

## 6. 测试策略 (Validation)
*   **双人同步测试**: 验证 A 答题后 B 是否能即时感知（Watcher 延迟测试）。
*   **离线补偿测试**: 模拟弱网环境下心跳共振的补发逻辑。
*   **UI 视觉检查**: 确保在不同设备分辨率下，纸张纹理和排版不产生畸变。
